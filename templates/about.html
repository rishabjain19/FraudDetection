About This Project

Purpose

The Fraud Detection System for Banking Transactions is a comprehensive Database Management System (DBMS) project designed to automatically identify and flag suspicious banking transactions in real-time. This system helps financial institutions protect their customers and prevent fraudulent activities through intelligent database triggers and automated monitoring.

Real-World Use Cases

Banking & Financial Services

Monitor and prevent fraudulent credit/debit card transactions, unauthorized transfers, and suspicious account activities.

Payment Processing

Real-time fraud detection for online payment gateways, e-commerce platforms, and digital wallets.

Security & Compliance

Ensure regulatory compliance with financial transaction monitoring requirements (KYC, AML policies).

Analytics & Reporting

Generate insights on transaction patterns, identify potential security threats, and improve fraud prevention strategies.

DBMS Concepts Implemented

1. CRUD Operations

  • CREATE: Insert new transactions via Add Transaction form
  • READ: View all transactions and fraud alerts
  • UPDATE: Edit existing transaction details
  • DELETE: Remove transactions from the database

2. Joins

  • LEFT JOIN used in vw_suspicious view to combine fraud_alerts with transactions table
  • Provides comprehensive information by linking related records

3. Aggregate Functions

  • COUNT(*) used in triggers to count rapid transactions
  • Aggregate calculations for fraud detection logic

4. Subqueries

  • Nested SELECT queries in triggers to check transaction patterns
  • Subqueries validate conditions before inserting fraud alerts

5. Views

  • vw_suspicious view combines fraud_alerts and transactions
  • Simplifies complex queries and provides ready-to-use data
  • Used in Fraud Alerts page for displaying comprehensive information

6. Indexes

  • Index on user_id for faster user-based queries
  • Index on txn_time for efficient time-based searches
  • Composite index on (user_id, txn_time) for rapid transaction detection

7. Triggers

  • trg_flag_high_amount: Automatically flags transactions exceeding ₹10,000
  • trg_flag_rapid_txns: Detects more than 5 transactions within 5 minutes for the same user
  • Triggers execute automatically after INSERT operations

8. Stored Procedures (Conceptual)

  • Trigger logic acts as stored procedures in SQLite
  • Encapsulates business logic at database level
  • Ensures data integrity and automated fraud detection

Technical Stack

Backend: Flask (Python)
Database: SQLite3
Frontend: HTML5, CSS3
